# 产品方案

## 产品名称（暂定）：AgentOS

---

# 1. 产品概述

AgentOS 是一个本地运行的 Agent 操作系统。
其核心目标是：

* 将本地文件系统转换为 AI 可操作的知识空间
* 以对话作为唯一主要交互方式
* 支持 Agent 对文件进行查询、修改和生成

产品不替代文件系统，而是在文件系统之上构建一层 Agent 操作抽象。

---

# 2. 产品目标

## 2.1 核心目标

1. 用户不再通过“打开文件”获取信息
2. 用户通过对话完成知识查询和修改
3. 文件作为底层数据存在，但不再是主要交互界面

## 2.2 MVP目标

* 支持本地文件目录接入
* 支持自动转换为 Markdown
* 支持 Agent 查询
* 支持 Agent 修改
* 支持 Agent 创建文件
* 回答中可标注来源

---

# 3. 产品范围

## 3.1 支持文件类型

* .docx
* .pdf
* .pptx
* .xlsx
* .md
* .txt

所有文件统一转换为 Markdown 镜像文件。

---

## 3.2 不支持功能（MVP阶段）

* 多人协作
* 实时编辑
* 评论系统
* 权限管理
* 富文本编辑器
* 表格可视化编辑
* 云同步

---

# 4. 核心功能设计

---

## 4.1 Workspace 管理

### 功能

* 用户选择一个本地目录
* 系统创建对应的 Agent 镜像目录
* 支持多个 workspace
* 每个 workspace 独立运行

### 目录结构

原始文件目录：

```
/user-workspaces/project-a
```

镜像目录：

```
/agent-space/project-a/docs
```

---

## 4.2 文件转换机制

### 功能

* 扫描原始目录
* 调用转换工具（如 MarkItDown）
* 将文件转为 Markdown
* 保留目录结构
* 生成镜像文件

### 原则

* 原文件不修改
* Markdown 作为 AI 镜像层
* 镜像文件可被 Agent 修改

---

## 4.3 搜索机制（MVP）

使用 grep 进行文本搜索。

### 搜索流程

1. 从用户问题中提取关键词
2. 对镜像目录执行 grep
3. 获取匹配片段
4. 提供给 Agent 进行综合回答

---

## 4.4 Agent 查询流程

### 流程

1. 用户输入问题
2. Agent 判断是否需要调用工具
3. 调用 search_files
4. 如需要调用 read_file
5. 基于返回内容生成回答
6. 如引用文件内容，标明来源

---

## 4.5 Agent 修改流程

### 流程

1. 用户输入修改指令
2. Agent 搜索并读取目标文件
3. 生成完整修改后 Markdown
4. 自动备份原文件
5. 覆盖写入
6. 返回修改结果

---

## 4.6 Agent 创建文件

### 流程

1. 用户提出生成需求
2. Agent 生成 Markdown
3. 创建新文件
4. 返回文件路径

---

# 5. 工具系统设计

Agent 通过工具访问文件系统。

## 工具列表

1. list_files
2. search_files
3. read_file
4. write_file
5. create_file

Agent 不直接读取所有文件内容。

---

# 6. 回答来源规则

* 若回答引用文件内容，必须标明来源
* 若基于通用知识，可不标注来源
* 禁止伪造来源
* 修改文件时必须标明文件路径

示例：

```
违约金比例为5%。

[来源: project-a/contract.md:120]
```
