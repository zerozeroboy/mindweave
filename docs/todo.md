# MindWeave Todo

## 任务清单

### P1

- [ ] 实现实时打断与纠偏（Human-in-the-loop）
  - 背景/问题
    - 针对长任务（如大范围查阅代码或整理长文档），当前 `agent-runtime.ts` 仅有被动的超时熔断（90秒）和轮次限制（12轮）。
    - 缺少实时干预手段，当 Agent 方向错误时，用户只能干等熔断，浪费时间和 Token。
  - 目标
    - 在对话过程中提供明确的“停止并介入”机制，支持切断网络请求和后台循环，节省成本并支持及时纠偏。
  - 范围
    - 前端（UI层）：增加打断按钮交互和状态管理。
    - API层：引入请求级别的可中止信号（如 `AbortController` / SSE 连接断开监听）。
    - 运行时层（`agent-runtime.ts`）：支持传递和响应中止信号（`AbortSignal`），在循环边界与大模型请求层进行熔断。
  - 方案建议
    - 前端发起 `/api/chat` 请求时绑定 `AbortController`，并在用户点击“停止生成”时调用 `abort()` 中断 SSE 流。
    - 后端接口监听客户端断开连接，更新对应对话的取消标志。
    - 运行时 `runAgentChatStream` 的 `while(true)` 循环节点以及调用 `doubao-client` 时均判断中断标志，如果中断则立刻 yield 中止系统提示并平滑退出。
  - 验收标准
    - 在 Agent 连续执行工具期间，用户点击“停止”，Agent 能立刻停止后续工具调用与输出。
    - 输出明确的系统提示（如 `[系统提示：用户已手动终止]`）。
    - 底层调用正常取消，不会产生额外的 API 调用开销。
